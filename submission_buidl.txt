SUMPLUS TRADER. BNB Hack BUIDL submission text (paste-ready)

=== PROJECT NAME ===
Sumplus Trader

=== TAGLINE ===
A self-custody AI trader on BSC that can prove it stayed inside its mandate, running unattended for a week.

=== SHORT DESCRIPTION ===
Sumplus Trader is an autonomous on-chain trading agent built on the official stack: Trust Wallet Agent Kit for self-custody signing, CoinMarketCap MCP for data, and ERC-8004 for agent identity. The thing that sets it apart is that you can verify it. Before the market opens, the agent publishes its policy hash on-chain. For the whole week, every decision it makes is a hash-chained receipt that references that commitment. After the window closes, anyone can recompute the hash from the public repo and confirm the agent traded by rules that were fixed before the market moved. Rule adherence is something a judge can check, not something they have to take on faith.

=== FULL DESCRIPTION ===
In autonomous on-chain trading the strategy is rarely the hard part. The hard part is trust: proving an agent stayed inside its mandate when no human was watching. That is the problem Sumplus Trader is built to solve, and it is what this hackathon exists to advance.

Three-layer trust stack:
1. Trust Wallet Agent Kit signs every transaction with self-custody. Keys stay local. The agent is registered on-chain via twak compete register.
2. ERC-8004 gives the agent a self-sovereign on-chain identity. We publish the policy hash to it before code-lock, a commit-reveal that timestamps our rules on-chain.
3. Maria, our verifiable decision layer, writes a tamper-evident, hash-chained receipt for every decision the agent makes, including the ones where it decides not to trade. Break any receipt and every hash after it breaks.

Data comes only from the CoinMarketCap MCP server, and every fetch logs an x402 receipt as paid-data provenance.

How it trades: survival first. The competition eliminates any agent that breaches roughly 6 percent drawdown, and it scores risk-adjusted return alongside rule adherence. So the agent is built to stay disciplined and never get eliminated. It takes risk only when 1h and 4h momentum agree and volatility is low, caps risky exposure at 12 percent, and runs a drawdown ladder that de-risks ahead of the gate, flattening to stablecoins at 3 percent. In a six-week stress test that includes a severe crash, the committed live strategy never breaches the gate, with a peak drawdown near 2.8 percent and a receipt chain that stays fully intact.

The agent treats a decision not to trade as a real decision. Every skip is a reasoned receipt, marked to market later in an avoided-loss ledger, so restraint shows up in the record instead of disappearing.

The whole thing runs unattended and is ops-hardened: a watchdog restarts the loop on any crash, state is persisted atomically so a restart resumes exactly, RPC endpoints fail over, and the agent is fail-closed, stopping trading rather than acting on stale or divergent data.

A live dashboard shows the committed hash with a verified badge, NAV and a drawdown gauge against the gate, the equity curve, a streaming decision feed, and a black-box replay that recomputes any receipt's hash on demand.

The whole agent runs offline with no keys: a mock brain, a mock backend, and a deterministic CoinMarketCap scenario. Judges can clone it and watch the verifiable trail build, or just open the hosted dashboard.

=== TRACK 1 (Autonomous Trading Agents) ===
The live agent. Self-funded wallet on BSC, registered before the window, trading PancakeSwap unattended 6/22 to 6/28. Survival-first strategy, verifiable receipts, on-chain policy commitment. Wallet, identity, and commit-reveal transaction are linked from the dashboard and config/proof.json.

=== TRACK 2 (Strategy research) ===
A backtestable head-to-head: the live committed strategy (champion) against a naive DCA baseline (challenger) across a regime-rich synthetic series. The champion survives all six weeks at 3.93 percent peak drawdown, well inside the 6 percent gate, and finishes +7.58 percent. The challenger is eliminated in the crash at hour 564 as its drawdown blows past the gate to 29 percent. On the window where both are alive, champion Sharpe is 1.67 versus the challenger's 0.87 and champion Calmar is 0.47 versus 0.12. The challenger's flattering +34 percent full-period return is a mirage the elimination gate erases. Full write-up and reproducible code in the repo. Run: python backtest/run.py
